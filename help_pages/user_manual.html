🎯 What is this tool?

Query your database using natural language - no SQL knowledge required!

❓ Common Questions

Click on any question to see the answer

How do I limit the number of results? +
Just specify in your question: "Show me the top 10 transactions" or "Give me the first 20 customers"
Can I modify or delete data? +
No, this tool is read-only for safety. It only retrieves data, never modifies it.
What if my query is taking too long? +
Queries have automatic timeouts (default: 30 seconds). You can adjust this in Settings if needed.
How do I know what tables/columns are available? +
Use the Schema Browser tab to search for tables and columns in your database.
Can I see the actual SQL that was generated? +
Yes! Go to the Inspect tab and check the Generated SQL section.
What are embeddings and why do they matter? +
Embeddings are a smart way the system learns about your database structure. On first startup, the system creates embeddings (takes 30-60 seconds), but subsequent startups are much faster (2-5 seconds) because it reuses them.
When should I refresh embeddings? +
Refresh embeddings when:
  • You've added new tables or columns
  • Column values have changed significantly
  • The system isn't finding the right tables
What is the Agent Recursion Limit? +
It controls how many reasoning steps the AI agent can take. Higher limits allow more complex reasoning but take longer:
  • 25 (Default): Good for most queries
  • 50: For complex multi-table queries
  • 75: For very complex analytical questions
  • 100: Maximum - use only for extremely difficult questions

📊 Managing Embeddings (Advanced)

Understand and control how the system learns your database

What Gets Embedded?

1. Table Names
Names and descriptions of all tables
2. Column Names
Column names and data types
3. Relationships
Foreign key connections
4. Distinct Values
Sample values from text columns

🎯 Refresh Scenarios

When and how to refresh embeddings

📋

Scenario 1: Added New Tables

What to refresh: Tables + Columns

  • Go to Settings → Embedding Management
  • Check: ☑️ Table Names, ☑️ Column Names
  • Uncheck: ☐ Relationships, ☐ Distinct Values
  • Click "Refresh Selected Types"
Takes only seconds!
💾

Scenario 2: Data Values Changed

What to refresh: Distinct Values only

  • Go to Settings → Embedding Management
  • Uncheck all except: ☑️ Distinct Values
  • Click "Refresh Selected Types"
  • Re-samples column values without touching schema
Smart refresh
🔗

Scenario 3: New Foreign Keys

What to refresh: Relationships only

  • Check only: ☑️ Relationships (FKs)
  • Click "Refresh Selected Types"
  • Fast FK relationship re-embedding
Quick update
🔄

Scenario 4: Major Changes

What to refresh: Everything

  • Click "Refresh Embeddings (Force Re-embed)" at top
  • Or check all four boxes
  • Complete schema re-learning
Full refresh

⚡ Performance Tips

  • First startup: Takes 30-60 seconds (creates embeddings)
  • Subsequent startups: Takes 2-5 seconds (reuses embeddings)
  • Selective refresh: Faster than full refresh
  • Embeddings persist: No need to refresh unless schema changes
