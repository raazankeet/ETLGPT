🛠️ Technical Documentation

Production-ready natural language to SQL translation system

A production-ready chatbot that translates natural-language questions into T-SQL for Microsoft SQL Server. Uses ChromaDB for semantic search, LangGraph ReAct agents for intelligent query planning, and provides comprehensive safeguards for safe query execution.

🏗️ System Architecture

Core components and data flow

💾

Core Components

  • Database: Microsoft SQL Server
  • Embedding Store: ChromaDB
  • Embedding Model: SentenceTransformers
  • Agent Framework: LangGraph ReAct
  • UI Framework: Gradio
  • LLM: OpenAI API
🔄

Data Flow

  • User submits natural language query
  • ReAct agent analyzes and plans
  • Consults semantic embeddings
  • SQL generated with validation
  • Query executes with safeguards
  • Results formatted and returned
  • Full trace logged with UUID

⚙️ Configuration Files

Essential configuration and setup

📄 .env (Required)

Database connection parameters:

MSSQL_SERVER=your-server
MSSQL_DATABASE=your-database
MSSQL_USER=your-username
MSSQL_PASSWORD=your-password
MSSQL_DRIVER=ODBC Driver 18 for SQL Server  # Optional
MSSQL_PORT=1433  # Optional
MSSQL_ENCRYPT=yes  # Optional
OPENAI_API_KEY=your-openai-key

📄 config/embedding_config.yaml (Optional)

Control semantic embeddings and schema:

descriptions:
  TableName: "Human-readable table description"
  
skip_tables:
  - temp_table
  - system_table
  
embed_values:
  TableName:
    - column1
    - column2
    
skip_values:
  - TableName.sensitive_column
  
safeguards:
  default_row_limit: 10
  query_timeout_seconds: 30

📄 config/prefs.json (Auto-generated)

Runtime preferences managed via Settings tab:

  • default_row_limit: Default TOP N for queries
  • query_timeout_seconds: Query execution timeout
  • chroma_only: Skip live DB value lookups
  • stream_trace: Enable live agent trace streaming
  • ui_density: UI spacing/text size
  • generate_only: Generate SQL without execution

🔄 Query Lifecycle

How queries are processed from input to output

1️⃣

Query Planning

  • Agent receives natural language input
  • Plans execution steps using tools
  • Iteratively refines approach
2️⃣

Schema Discovery

  • search_tables: Find relevant tables
  • search_columns: Match query intent
  • search_relationships: Discover FKs
  • search_values: Find example values
3️⃣

Value Resolution

  • Checks ChromaDB embeddings first
  • Falls back to live DISTINCT queries
  • Caches results for future queries
4️⃣

SQL Generation

  • Constructs T-SQL using context
  • Validates syntax with SET PARSEONLY
  • Retries on errors (up to 3 attempts)
5️⃣

Execution & Safeguards

  • Injects TOP N if not specified
  • Enforces query timeout
  • Logs all attempts and final SQL
  • Records metadata (time, duration, rows)
6️⃣

Result Formatting

  • Returns natural language summary
  • Provides structured table data
  • Includes full audit trail

📊 Embedding Strategy

What gets embedded and how

What Gets Embedded?

Tables (always)
Table names and descriptions from config
Columns (always)
Column names, data types, Table.Column format
Relationships (always)
Foreign key constraints
Values (selective)
Only columns listed in embed_values

⚡ Bootstrap Process

Run on first launch or manually via Settings:

  • 1. Discovers all tables/columns via INFORMATION_SCHEMA
  • 2. Excludes tables in skip_tables
  • 3. Embeds table/column metadata
  • 4. Samples values from embed_values columns
  • 5. Stores embeddings in ChromaDB (./data/chroma_db/)

🛡️ Safeguards & Security

Built-in protections and security features

🔒

Query Safety

  • SELECT-only: Agent generates only SELECT
  • Row Limits: Automatic TOP N injection
  • Timeouts: Queries terminated after threshold
  • Syntax Validation: Pre-execution parsing
🔐

Data Privacy

  • Credentials stored locally in .env
  • No external data transmission (except LLM)
  • Embedding cache persists locally
  • Query history bounded to 1000 entries
  • Comprehensive logging with UTF-8
⚙️

Agent Configuration

  • Recursion Limit: Max reasoning steps (default: 25)
  • Adjustable: 25, 50, 75, or 100 steps
  • Higher values = more complex reasoning
  • Stored in user preferences
👤

Access Control

  • Inherits SQL Server user permissions
  • No privilege escalation
  • Read-only operation mode

🔑 API Key Protection

  • • Rotate keys if exposed
  • • .gitignore excludes .env by default

🔍 Troubleshooting

Common issues and solutions

Slow Startup Every Time +

Symptoms: Application takes 30-60 seconds to start every time

Cause: Embeddings are being re-created on each startup

Solutions:

  • Check if data/chroma_db/ directory exists and has data
  • Verify ChromaDB collections are accessible
  • Check logs for "Using existing embeddings" vs "Embeddings created"
  • Try deleting chroma_db/ and letting it rebuild once
System Can't Find Tables/Columns +

Symptoms: Agent says "table not found" or generates incorrect SQL

Cause: Embeddings may be stale or missing

Solutions:

  • Go to Settings → Embedding Management
  • Click "View Current Embeddings" to verify tables
  • Use "Refresh Embeddings (Force Re-embed)"
  • Check config for skip_tables entries
Wrong Column Values in Queries +

Symptoms: Agent uses incorrect values in WHERE clauses

Cause: Distinct value embeddings are outdated

Solutions:

  • Go to Settings → Embedding Management → Selective Refresh
  • Check only "Distinct Values"
  • Click "Refresh Selected Types"
  • Verify correct columns in embed_values in config
Agent Stopped Due to Iteration Limit +

Symptoms: Error message or incomplete answers

Cause: Complex question requires more reasoning steps

Solutions:

  • Increase recursion limit: Settings → Agent Recursion Limit
  • Use 50 for complex multi-table queries
  • Use 75 for very complex analytical questions
  • Use 100 for extremely difficult questions
  • Simplify question into smaller queries
Can't Connect to SQL Server +

Symptoms: "ODBC connection failed" error

Solutions:

  • Verify .env file has correct credentials
  • Test network connectivity to SQL Server
  • Check firewall rules (port 1433)
  • Verify ODBC driver is installed
  • Try adding ,1433 port explicitly to server name
  • Test with Settings → Test Connection
Queries Taking Too Long +

Symptoms: Timeout errors or slow responses

Solutions:

  • Increase timeout: Settings → Query Timeout
  • Check if query has proper indexes
  • Review generated SQL in Inspect tab
  • Consider adding TOP N to limit results
  • Enable "Chroma-only mode" to skip live lookups

⚙️ Advanced Configuration

Fine-tune system behavior

📐

UI Density

Adjust text size and spacing for different displays:

  • XS: Minimal spacing, compact
  • S: Small (default)
  • M: Medium
  • L: Large, comfortable
  • XL: Extra large, accessibility
⚡

Chroma-Only Mode

Skips live database value lookups, uses only cached embeddings.

Benefits: Faster, reduced DB load

Drawbacks: May miss recent values

🔍

Generate-Only Mode

Creates SQL without executing it.

Use case: Review SQL before running, testing, development

📊

Stream Trace

Live-streams agent reasoning steps during query execution.

Benefits: Real-time debugging, transparency

Performance: Minimal overhead

🔌 API Surface

Tools available to the agent

Available Tools

  • search_tables(query, top_k) - Semantic table search
  • search_columns(query, top_k) - Semantic column search
  • search_relationships(query, top_k) - Find FK relationships
  • search_values(query, top_k) - Find example values
  • fetch_distinct_values(table, column, limit) - Live value lookup
  • validate_sql(sql) - Syntax check via SET PARSEONLY
  • execute_sql(sql) - Run SELECT query with safeguards
